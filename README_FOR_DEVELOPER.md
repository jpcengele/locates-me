## üìê Architecture

### Core Logic (`backend/core/engine.py`)
The heart of the system. It converts coordinates to/from 3-word addresses.
1.  **Geo to H3:** `lat, lon` ‚Üí `h3_index` (64-bit integer)
2.  **Scramble:** `h3_index` ‚Üí `encrypted_int` (using FF1 with a secret key)
3.  **Encoding:** `encrypted_int` ‚Üí `Word.Word.Word` (Base-N encoding using language vocabulary)

### Database (`backend/locates.db`)
Used **ONLY** for storing "Sovereign Plate" claims. The core addressing system does **not** use the database.

## ‚ùì FAQ

### Why H3 instead of Squares (What3Words)?
*   **Topological Uniformity:** Hexagons have uniform distance to all neighbors, making spatial analysis superior.
*   **Low Distortion:** H3 minimizes distortion across the globe compared to square grids.

### Is this compatible with What3Words?
**No.** This is a sovereign, open-source alternative. It does not rely on W3W's proprietary database.
